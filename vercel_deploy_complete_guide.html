Complete step-by-step guide to designing and deploying a web app on Vercel

1. Setup
2. Build app
3. GitHub
4. Vercel
5. Live!
Phase 1 — Setup tools

Install the tools you need

You need Node.js, VS Code, and a GitHub account. All free, no card required.

1
Install Node.js
Go to nodejs.org → download the LTS version → install it. This gives you npm (package manager).
Verify installation — open your terminal and run:node --version npm --version
You should see version numbers like v20.x.x — that means Node is installed correctly.
2
Install VS Code
Go to code.visualstudio.com → download → install. This is your code editor.
3
Create a GitHub account
Go to github.com → Sign up → choose a username and password. No card needed.
Remember your GitHub username — you'll need it in Step 3.
4
Install Git
Go to git-scm.com → download → install. Then set your identity in the terminal:
git config --global user.name "Your Name" git config --global user.email "your@email.com"
Phase 2 — Design your web app

Create a React app using Vite

We'll scaffold a real project in under 2 minutes using the terminal.

1
Open your terminal
On Windows: press Win + R → type cmd → Enter. On Mac: Cmd + Space → type Terminal.
2
Scaffold the project
Run these commands one by one:
npm create vite@latest my-portfolio -- --template react cd my-portfolio npm install
Vite is a fast build tool. It creates a ready-to-go React project structure for you automatically.
3
Open in VS Code
code .
This opens the entire project folder in VS Code. You'll see this file structure:
my-portfolio/ ├── src/ │ ├── App.jsx ← your main page │ ├── App.css ← your styles │ └── main.jsx ├── index.html └── package.json
4
Edit App.jsx — customise your app
Open src/App.jsx and replace its contents with this simple portfolio page:
function App() { return ( <div style={{textAlign:'center', padding:'50px', fontFamily:'sans-serif'}}> <h1>My Web App</h1> <p>Deployed on Vercel using PaaS!</p> <p style={{color:'#888'}}>Built with React + Vite</p> </div> ) } export default App
5
Run locally to test
npm run dev
Open http://localhost:5173 in your browser. You should see your app running!
App works locally — ready to deploy!
6
Create the production build
npm run build
This creates a dist/ folder with your optimised app ready for deployment.
Phase 3 — Push to GitHub

Upload your code to GitHub

Vercel connects directly to GitHub. First get your code on GitHub, then Vercel can pull it.

1
Create a new repository on GitHub
Go to github.com → click the + icon (top right) → New repository.
Fill in the form:
›Repository name: my-portfolio
›Visibility: Public
›Do NOT tick "Add README" (keep it empty)
›Click Create repository
2
Initialise Git in your project
In your terminal (inside the my-portfolio folder):
git init git add . git commit -m "first commit"
3
Connect and push to GitHub
Copy your repo URL from GitHub (looks like the one below) and run:
git branch -M main git remote add origin https://github.com/YOUR_USERNAME/my-portfolio.git git push -u origin main
Replace YOUR_USERNAME with your actual GitHub username!
4
Verify on GitHub
Go to your GitHub profile → click your repository. You should see all your project files listed there.
Code is on GitHub — ready to connect Vercel!
Phase 4 — Deploy on Vercel

Connect GitHub to Vercel and deploy

No card needed. Just your GitHub account. The whole thing takes about 2 minutes.

1
Sign up on Vercel
Go to vercel.com → click Sign Up → choose Continue with GitHub → authorise Vercel.
No credit card is asked — Vercel only needs your GitHub login.
2
Import your project
After login you land on the dashboard. Click Add New Project.
On the "Import Git Repository" screen:
›You'll see your GitHub repos listed
›Find my-portfolio → click Import
3
Configure build settings
Vercel auto-detects Vite + React. Confirm these settings (they should already be filled):
Build settings (auto-detected):
Framework Preset: Vite
Build Command: npm run build
Output Directory: dist
Install Command: npm install
4
Click Deploy!
Click the big Deploy button. Vercel will run your build and deploy it to their global CDN.
You'll see the build log in real time:
Cloning repository...
Installing dependencies...
Running build command: npm run build
Build completed in 12s
Deploying to production...
Build usually completes in 30–90 seconds.
Phase 5 — Your app is live!

Your web app is deployed on the internet

Vercel gives you a live URL instantly. Share it with anyone.

1
Get your live URL
After deploy, Vercel shows a success screen with your public URL:
https://my-portfolio-abc123.vercel.app
Your web app is live and accessible to anyone in the world!
2
Test CI/CD — automatic redeployment
Edit something in App.jsx (change the heading), save, then push:
git add . git commit -m "updated heading" git push
Watch your Vercel dashboard — it automatically starts a new build! This is the power of PaaS CI/CD.
3
View deployments dashboard
In Vercel dashboard → click your project → Deployments tab. You'll see every deployment with its status, time, and build logs.
Deployments list:
Production — updated heading — 2 min ago
Production — first commit — 15 min ago
4
What you've accomplished
PaaS concepts demonstrated:
Designed a web app locally
Used version control (Git + GitHub)
Deployed to a PaaS platform (Vercel)
Used CI/CD — auto rebuild on push
App hosted on a global CDN
Zero cost, zero card required
